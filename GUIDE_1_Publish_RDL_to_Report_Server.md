# Publishing RDL Reports to Power BI Report Server (On-Prem)

## Complete Step-by-Step Guide

This guide takes you from the `.rdl` files generated by the migration tool
all the way to a live published report on your on-prem Power BI Report Server.


## Prerequisites

Before you begin, make sure you have:

1. **Power BI Report Server** installed and running on your organization's server
   - You need the web portal URL (e.g., `https://reportserver.yourcompany.com/reports`)
   - Ask your IT/infrastructure team if you don't know the URL

2. **Power BI Report Builder** installed on your Windows machine
   - Free download: https://www.microsoft.com/en-us/download/details.aspx?id=58158
   - This is different from "Power BI Desktop" — Report Builder is specifically for paginated (.rdl) reports

3. **Database access credentials** for the database your WebFocus reports were querying
   - Server name, database name, and authentication method
   - If you used Windows auth with WebFocus, you'll likely use the same with Report Server

4. **Permissions** on the Report Server
   - You need at least "Publisher" role on the target folder
   - Ask your Report Server admin to grant this if needed


## Step 1: Extract the RDL Bundle

After clicking "Generate RDL Bundle" in the migration tool, you'll download
a ZIP file called `webfocus_rdl_bundle.zip`.

Extract it. You'll see a folder structure like:

```
webfocus_rdl_bundle/
├── paginated/
│   ├── compensation_report.rdl
│   └── quarterly_summary.rdl
├── self-service/
│   └── infoassist_sales.rdl
├── standard/
│   ├── daily_report.rdl
│   └── regional_summary.rdl
├── active-reports/
│   └── interactive_dashboard.rdl
└── _MIGRATION_MANIFEST.txt
```

Open `_MIGRATION_MANIFEST.txt` first — it tells you the status of each file,
confidence scores, and any files that were skipped or had errors.


## Step 2: Open an RDL File in Report Builder

1. Launch **Power BI Report Builder** from your Start menu
2. Go to File → Open
3. Navigate to your extracted folder and select an `.rdl` file
4. The report opens in design view showing the layout


## Step 3: Configure the Data Source Connection

This is the most important step. The generated RDL has a placeholder
data source that you need to point to your actual database.

1. In the left panel, find **Data Sources** under the Report Data pane
   - If you don't see the Report Data pane, go to View → Report Data

2. Right-click on **WebFocusDataSource** → **Properties**

3. In the dialog that opens:

   a. **Name**: Keep as-is or rename (e.g., "ProductionDB")

   b. **Connection type**: Select your database type
      - "Microsoft SQL Server" for SQL Server
      - "Oracle" for Oracle databases
      - "ODBC" for ODBC connections (similar to what WebFocus used)

   c. **Connection string**: Enter your actual database connection
      Examples:

      SQL Server:
      ```
      Data Source=your-db-server.company.com;Initial Catalog=YourDatabase
      ```

      Oracle:
      ```
      Data Source=your-oracle-sid;User ID=username;Password=password
      ```

      ODBC:
      ```
      DSN=YourODBCDataSourceName
      ```

   d. **Credentials**: Click "Credentials" and choose:
      - "Use Windows Authentication" (if your org uses AD)
      - "Use this username and password" (enter DB credentials)
      - "Prompt for credentials" (users enter credentials at runtime)

4. Click **Test Connection** to verify it works
5. Click **OK** to save


## Step 4: Verify the Dataset Query

1. In the Report Data pane, expand **Datasets**
2. Right-click on the dataset (e.g., "DS_STORE_SALES_1") → **Properties**
3. Check the **Query** tab — you'll see the SQL generated from your FEX file

   Example:
   ```sql
   SELECT REGION_NAME, QUARTER_NAME, REVENUE, COST_OF_GOODS, GROSS_MARGIN
   FROM dbo.STORE_SALES
   WHERE FISCAL_YEAR = @FISCAL_YEAR
     AND REVENUE > 0
   ```

4. If the table name or column names are different in your actual database,
   edit them here

5. Click **Refresh Fields** to pull live column metadata from your database
6. Click **OK**


## Step 5: Check Report Parameters

If your WebFocus report had ACCEPT/PROMPT/-SET variables, they've been
converted to report parameters.

1. In the Report Data pane, expand **Parameters**
2. Right-click any parameter → **Properties**
3. Verify:
   - **Name** matches what the query expects (e.g., FISCAL_YEAR)
   - **Data type** is correct (String, Integer, Date, etc.)
   - **Default Values** tab has the right default
   - **Available Values** tab has a list of choices if applicable

4. For parameters that should show a dropdown of values:
   a. Create a new dataset that queries for valid values
      (e.g., SELECT DISTINCT FISCAL_YEAR FROM STORE_SALES)
   b. In the parameter properties → Available Values → "Get values from a query"
   c. Select the dataset, value field, and label field


## Step 6: Preview the Report

1. Click the **Run** button in the toolbar (or press F5)
2. If you have parameters, you'll see a parameter bar at the top
3. Enter/select parameter values and click **View Report**
4. The report renders with live data from your database
5. Check:
   - Data is showing correctly
   - Column headers match your expectations
   - Group breaks and subtotals appear where expected
   - Page numbers show in the footer
   - Charts render properly (if your FEX had GRAPH FILE)

6. Click **Design** to go back to edit mode if you need to fix anything


## Step 7: Adjust Formatting (If Needed)

Common adjustments you might want to make:

**Column widths**: Click a column edge in the tablix and drag to resize

**Number formats**: Right-click a cell → Text Box Properties → Number
  - Select the format category (Number, Currency, Date, etc.)
  - Set decimal places

**Fonts and colors**: Right-click a cell → Text Box Properties → Font
  - Change font family, size, weight, color

**Conditional formatting**: Right-click a cell → Text Box Properties →
  Fill → click the expression button (fx) to add conditional colors:
  ```
  =IIF(Fields!MARGIN_PCT.Value > 20, "LightGreen",
   IIF(Fields!MARGIN_PCT.Value < 10, "LightCoral", "White"))
  ```

**Page size**: Go to Report → Report Properties → Page Setup
  - Change from Portrait to Landscape if you have many columns
  - Adjust margins


## Step 8: Save Locally (Optional but Recommended)

Before publishing, save a local copy:
1. File → Save As → select a local folder
2. This gives you a backup you can re-edit later


## Step 9: Publish to Power BI Report Server

1. Go to File → **Save As** → **Report Server**

2. Enter your Report Server URL:
   ```
   https://reportserver.yourcompany.com/reports
   ```
   (This is the web portal URL — ask your IT team if unsure)

3. Click **Open** — you'll see the folder structure on the server

4. Navigate to the folder where you want to publish
   - Create a new folder if needed (right-click → New Folder)
   - Recommended structure:
     ```
     /Migrated Reports/
     ├── Sales/
     ├── HR/
     ├── Finance/
     └── Operations/
     ```

5. Enter a file name for the report (e.g., "Quarterly Sales Summary")

6. Click **Save**


## Step 10: Configure the Data Source on the Server

After publishing, the report's data source needs to be configured
on the server side too:

1. Open your browser and go to your Report Server web portal
   ```
   https://reportserver.yourcompany.com/reports
   ```

2. Navigate to the folder where you published the report

3. Click the **three dots (⋯)** on the report → **Manage**

4. Go to the **Data Sources** section

5. For each data source:
   a. Select "A shared data source" if your org has shared data sources set up
      (recommended — this way you configure the connection once and all
      reports reuse it)

   OR

   b. Select "A custom data source" and enter:
      - Connection type (SQL Server, Oracle, etc.)
      - Connection string
      - Credentials (stored credentials are recommended for scheduled reports)

6. Click **Apply**


## Step 11: Test the Published Report

1. In the web portal, click on your published report
2. It opens in the browser with a parameter bar at the top (if applicable)
3. Enter parameters and click **View Report**
4. Verify the report renders correctly with live data
5. Test the **Export** button — you can export to:
   - PDF
   - Excel
   - Word
   - PowerPoint
   - CSV


## Step 12: Set Up Shared Data Sources (For Batch Publishing)

If you're publishing 100+ reports, create shared data sources first
so you don't configure connections 100+ times:

1. In the web portal, go to the root or a shared folder
2. Click **+ New** → **Data Source**
3. Name it (e.g., "ProductionDB")
4. Configure connection type, string, and credentials
5. Save

Then for each published report:
1. Manage → Data Sources → change to "A shared data source"
2. Browse to your shared data source
3. Apply

This way, if your database server changes, you update ONE shared
data source and all 100+ reports automatically use the new connection.


## Step 13: Set Up Subscriptions (Optional)

If you want reports to run on a schedule and email the results:

1. Open a report in the web portal
2. Click **Subscribe** (in the toolbar)
3. Configure:
   - **Delivery method**: Email, File Share, or Null
   - **Schedule**: Daily, Weekly, Monthly, or Custom
   - **Parameters**: Set the parameter values for the scheduled run
   - **Rendering format**: PDF, Excel, etc.
4. Save the subscription

The report runs automatically and delivers results to the configured
destination.


## Troubleshooting

**"Cannot connect to data source"**
→ Check the connection string in Step 3. Verify the database server
  is reachable from the Report Server machine (not just your desktop).

**"Parameter @FISCAL_YEAR is missing"**
→ The SQL query references a parameter that wasn't created.
  Add it manually in Report Data → Parameters → right-click → Add Parameter.

**"Column not found"**
→ Your actual database column names differ from what WebFocus used.
  Edit the dataset query (Step 4) to use the correct column names.

**Report shows no data**
→ Run the SQL query directly in SQL Server Management Studio / your DB tool
  to verify it returns data with the parameter values you're using.

**Page breaks not working**
→ Check the group properties: right-click the row group in the tablix →
  Group Properties → Page Breaks → check "Between each instance of a group".

**"You do not have permission"**
→ Ask your Report Server administrator to grant you Publisher or
  Content Manager role on the target folder.


## Quick Reference: Publishing Checklist per Report

For each .rdl file in your bundle:

  [ ] Open in Report Builder
  [ ] Update data source connection (Step 3)
  [ ] Verify dataset query matches your DB schema (Step 4)
  [ ] Check parameters have correct defaults (Step 5)
  [ ] Preview with Run button (Step 6)
  [ ] Adjust formatting if needed (Step 7)
  [ ] Publish to Report Server (Step 9)
  [ ] Configure server-side data source (Step 10 — or use shared)
  [ ] Test in web portal (Step 11)
